# flow-prompt 스킬 사용법

한국어 장면 묘사를 Google 공식 5형식 영문 Flow 프롬프트로 바꾸는 스킬의 사용 안내다.

## 이 스킬이 하는 일

한국어로 장면을 묘사하면 Google 공식 5형식(Cinematography + Subject + Action + Context + Style & Ambiance)에 맞춘 영문 Flow 프롬프트를 만들어 준다. 아바타·인물이 말하는 립싱크 영상에 대사·음성을 넣는 작업에 맞춰져 있다.

## 언제 발동하나

"Flow 프롬프트", "Veo 프롬프트", "아바타 영상 프롬프트", "립싱크 영상 프롬프트", "AI 영상 프롬프트" 같은 말이나, 장면을 묘사하고 "영상으로 만들고 싶다"고만 해도 발동한다.

## 기본 사용 흐름

1. 만들고 싶은 장면을 한국어로 묘사한다. 예: "30대 여성 강사가 카메라 보고 인사하는 8초 세로 영상."
2. 스킬이 빠진 정보를 한 번에 묻는다. 아래 6개 중 안 준 것만 질문한다.
3. 답하면 5형식 프롬프트를 두 형태로 준다. 라벨 스캐폴드와 산문, 그리고 한국어 해설.
4. 산문형을 Flow 프롬프트 상자에 붙여 넣는다.

## 입력할 때 챙길 6가지

| 항목 | 내용 |
|------|------|
| 주체 | 아바타 / 실사 인물 / 캐릭터. 성별·연령·외형 |
| 화면비 | 가로(16:9) / 세로(9:16) |
| 길이 | Veo는 4·6·8초, Omni Flash는 4·6·8·10초 |
| 대사 | 있으면 정확한 문장 그대로 |
| 음성 톤 | 성별·연령·분위기(warm/energetic/calm 등) |
| 분위기·장소 | 배경, 시간대, 영상 톤 |

이 6개를 처음부터 다 주면 질문 없이 바로 프롬프트가 나온다. "질문 없이 바로 만들어 줘"라고 해도 건너뛴다.

## 모델 고르기

- 새 클립을 텍스트로 만든다 → Veo 3.1
- 기존 영상을 고친다 / 10초 클립 / 커스텀 음성 / 음성 참조 → Gemini Omni Flash

어느 모델을 전제로 만들었는지는 해설에 적혀 나온다.

## 받은 출력 읽는 법

- 라벨 스캐폴드: 항목별로 끊어 놓은 작업용 초안. 고치고 싶은 항목만 손본다.
- 산문(Flow 입력용): 라벨을 뗀 한 문단. 이걸 Flow에 붙여 넣는다.
- 해설: 가정한 값, Flow 설정 권장값(길이·화면비·모델), 대사·음성 처리 방식.

## Flow 화면에서 참조 거는 법

프롬프트에 들어가는 `@` 멘션은 Flow 프롬프트 상자에서 실제로 작동한다.

- `@me` — 본인 아바타(EEA·영국·스위스 미지원)
- `@캐릭터명` — 참고 이미지로 만든 캐릭터
- `@Voice: 이름` — 음성 참조(Omni Flash + 소재 방식에서만)
- 시작/종료 프레임·소재는 모델 → 동영상 → 프레임 / 소재에서 넣는다

## 짧은 예시

입력: "내 아바타가 카메라 보고 '안녕하세요, 오늘 영상을 시작합니다' 하는 8초 가로 영상. 따뜻한 목소리."

산문 출력:

```
Medium close-up, eye-level, static camera. @me looks directly into the camera and
speaks warmly with a slight smile, at a bright home desk by a side window in daytime.
Clean friendly tutorial look, soft daylight, warm tone. A warm, calm voice says,
"안녕하세요, 오늘 영상을 시작합니다." SFX: quiet room ambience. No on-screen text or subtitles.
```

## 자주 막히는 지점

- 대사는 반드시 따옴표 안에. 안 그러면 립싱크가 깨진다.
- 한 클립에 대사 한 문장. 길면 샷을 나눠 장면 빌더로 잇는다.
- 음성 참조(`@Voice`)는 소재 방식 영상에서만 된다. 프레임 방식에 넣으면 오류.
- 10초가 필요하면 Veo가 아니라 Omni Flash를 고른다.
- 해상도·버전 같은 수치가 필요하면 support.google.com/flow에서 다시 확인한다.

## Gemini CLI에서 쓰기

Gemini CLI는 Anthropic과 같은 SKILL.md 기반 Agent Skills(agentskills.io 오픈 표준)를 지원한다. 출처: geminicli.com/docs/cli/skills, geminicli.com/docs/cli/creating-skills. 이 스킬은 frontmatter가 `name`·`description` 표준이라 수정 없이 그대로 쓴다.

### 배치 위치

둘 중 하나에 `flow-prompt/` 폴더를 통째로 둔다. SKILL.md가 폴더 루트에 있어야 한다.

- 내 계정 전역: `~/.gemini/skills/flow-prompt/` 또는 `~/.agents/skills/flow-prompt/`
- 프로젝트(팀 공유, git): `.gemini/skills/flow-prompt/` 또는 `.agents/skills/flow-prompt/`

`.agents/skills/`는 다른 AI 도구와 호환되는 별칭이다. Claude와 Gemini를 한 git 저장소에서 공유하려면 이 경로를 권한다. 같은 이름이 겹치면 워크스페이스가 사용자 경로보다 우선하고, 같은 계층에서는 `.agents/skills/`가 `.gemini/skills/`보다 우선한다.

### 설치

1. `flow-prompt/` 폴더를 위 경로 중 하나에 둔다.
2. Gemini CLI를 켜고 `/skills list` 로 flow-prompt가 잡히는지 확인한다.
3. 별도 디렉토리에서 개발 중이면 그 폴더에서 `gemini skills link .` 로 연결한다.
4. Git 저장소로 배포하면 `gemini skills install <url> --consent` 로 설치한다.

### 쓰는 법

세션에서 "Flow 프롬프트 만들어줘" 같은 요청을 하면 Gemini가 description을 보고 이 스킬 활성화를 묻는다(activate_skill). 승인하면 SKILL.md 지침과 폴더가 대화에 주입된다. name(flow-prompt)은 디렉토리명과 같아야 한다.
