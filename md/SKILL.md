# Flow 프롬프트 생성

Google Flow는 Veo 3.1과 Gemini Omni Flash 영상 모델을 감싼 AI 영화 제작 도구다(labs.google/fx/tools/flow). 텍스트 프롬프트로 4~10초 클립을 만들고, 장면 빌더에서 여러 클립을 이어 붙인다. 이 스킬은 사용자의 한국어 장면 묘사를 Google 공식 프롬프트 형식인 5형식의 영문 프롬프트로 바꾼다. 특히 아바타·인물이 말하는 립싱크 영상에 음성과 대사를 넣는 작업에 초점이 있다.

## 출처와 단정 원칙 (먼저 읽는다)

이 스킬의 사양·기능·프롬프트 형식은 다음 Google 1차 공식 문서를 근거로 한다. "공식"이라 부르는 내용은 이 출처들로 확인된 것만이다.

- Flow 공식 지원센터: support.google.com/flow (모델·기능·클립 길이·참조 문법·접근 조건)
- Google Cloud 공식 Veo 3.1 프롬프트 가이드: cloud.google.com/blog/products/ai-machine-learning/ultimate-prompting-guide-for-veo-3-1 (5형식)
- DeepMind 공식 Veo 프롬프트 가이드: deepmind.google/models/veo/prompt-guide (요소 권장)

출처로 확인 못 한 내용은 "공식"이라 적지 않는다. 스킬이 임의로 정한 작업 규약과 Google 공식 사양을 섞어 단정하지 않는다. 사양은 수시로 바뀌므로, 중요한 수치를 답에 쓸 때는 위 출처를 재확인한다.

## 프롬프트 기본 형식 — Google 공식 5형식

기본 출력은 Google Cloud 공식 가이드가 제시한 5형식이다.

```
[Cinematography] + [Subject] + [Action] + [Context] + [Style & Ambiance]
```

- Cinematography: 카메라 워크와 샷 구도 (medium shot, dolly in, eye-level 등)
- Subject: 주체·초점 대상. 외형을 형용사로 구체화한다
- Action: 주체의 동작
- Context: 환경·배경 요소
- Style & Ambiance: 전체 미감, 무드, 조명

대사·효과음·네거티브는 5형식의 별도 슬롯이 아니라 공식이 따로 다루는 기법이다. 공식 가이드 기준으로 처리한다.

- 대사: 따옴표로 적는다. 예: `A woman says, "We have to leave now."`
- 효과음(SFX): 구체적으로 묘사한다. 예: `SFX: thunder cracks in the distance.`
- 주변음(Ambient): 배경 사운드스케이프로 적는다. 예: `Ambient noise: quiet room hum.`
- 네거티브: "제외할 것"을 서술형으로 적는다. 예: 막연한 `no buildings`보다 `a desolate landscape with no buildings or roads`.

### 두 단계 출력

1. **라벨 스캐폴드** — 5형식 라벨에 정보를 배치한 작업용 초안. 사용자이 항목을 보고 수정하기 쉽다.
2. **산문 변환** — 공식 예시처럼 라벨을 떼고 한 문단의 촬영 지시문으로 잇는다. Flow에는 산문형을 넣는다.

공식 산문 예시: `Medium shot, a tired corporate worker, rubbing his temples in exhaustion, in front of a bulky 1980s computer in a cluttered office late at night. The scene is lit by the harsh fluorescent overhead lights and the green glow of the monochrome monitor. Retro aesthetic, shot as if on 1980s color film, slightly grainy.`

## 작업 순서

1. **생성 전 필수 질문** — 묘사에 빠진 항목을 먼저 묻는다. 추측해서 바로 만들지 않는다. 아래 "생성 전 질문"을 따른다.
2. **모델·워크플로우 선택** — 새 클립 생성이면 Veo 3.1, 기존 영상 편집·10초 클립·커스텀 음성이면 Gemini Omni Flash를 고른다. 이미지를 먼저 만들어 영상화하면 2단계 워크플로우로 만든다. 아래 기준을 따른다.
3. **장면 정보 정리** — 답변과 묘사를 합쳐 5형식 항목(카메라·주체·동작·배경·스타일&무드)에 배치하고, 대사·SFX·네거티브를 따로 정리한다.
4. **클립 길이 판단** — Veo 3.1은 4·6·8초, Omni Flash는 4·6·8·10초다. 묘사가 한 클립을 넘으면 샷을 나눈다.
5. **5형식 프롬프트 출력** — 라벨 스캐폴드와 산문 변환을 함께 낸다.
6. **한국어 해설 첨부** — 각 항목 의도, 가정한 값, Flow 설정 권장값(길이·화면비·모델)을 짧게 적는다. 사용자이 직접 수정할 수 있어야 한다.

## 생성 전 질문 (반드시 먼저 수행)

정보가 빠진 채로 만들면 원치 않은 영상이 나온다. 생성 전에 항상 빠진 항목을 묻는다. 다음 6개를 점검한다.

| 항목 | 묻는 내용 |
|------|-----------|
| 주체 | 아바타인지 실사 인물인지 캐릭터인지. 성별·연령대·외형 |
| 화면비 | 가로(16:9) / 세로(9:16) |
| 길이 | 클립 길이 (Veo 4·6·8초 / Omni Flash 4·6·8·10초) |
| 대사 | 대사 유무. 있다면 정확한 대사 문장 |
| 음성 톤 | 대사가 있을 때 — 성별·연령·분위기(warm/energetic/calm 등) |
| 분위기·장소 | 배경 장소, 시간대, 영상 톤 |

### 질문 방식

- 이미 확인된 항목은 다시 묻지 않는다. 빠진 항목만 한 번에 모아 묻는다.
- 답하기 쉽게 선택지를 함께 준다. 예: "화면비는 가로(16:9)인가요, 세로(9:16)인가요?"
- 카메라·조명·스타일은 비어도 합리적 기본값으로 채울 수 있다. 가정한 값은 해설에 밝힌다.
- 주체·화면비·길이·대사·음성 톤 5개가 빠지면 품질이 크게 흔들리므로 반드시 묻는다.

### 예외 — 바로 만들어도 되는 경우

6개 항목을 이미 충분히 묘사했거나 "질문 없이 바로 만들어 달라"고 하면 건너뛴다. 가정한 값은 해설에 모두 밝힌다.

## 모델 선택 — Veo 3.1 vs Gemini Omni Flash

출처: support.google.com/flow 모델·기능 문서. 둘은 역할이 다르다.

| 기능 | Veo 3.1 | Gemini Omni Flash |
|------|---------|-------------------|
| 텍스트→동영상 | 지원 (4·6·8초) | 지원 (4·6·8·10초) |
| 첫 프레임으로 동영상 | 지원 | 지원 |
| 첫+마지막 프레임 | 지원 | 지원 예정 |
| 소재(참고 자료) | Lite·Fast만, 8초 한정. Quality 미지원 | 지원, 고급 캐릭터·아바타·오디오 참고 |
| 기존 영상 AI 편집 | 미지원 | 지원 (최대 10초) |
| 연장 | 8초 한정 | 지원 예정 |
| 커스텀 음성 | 미지원 | 지원 |

기본값은 Veo 3.1이다. 다음이면 Omni Flash를 권한다: 기존 영상 수정, 10초 클립, 커스텀 음성, 음성 참조가 필요한 영상, 캐릭터·아바타·오디오 참고를 함께 쓰는 영상. 어느 모델을 전제로 만들었는지 해설에 밝힌다. Omni Flash는 모든 사용자에게 제공되며, 일부 기능은 특정 국가 한정이다.

## 참조 문법 (실제 Flow UI)

출처: support.google.com/flow 동영상·아바타 문서. Flow 프롬프트 상자에서 `@` 멘션으로 참조를 건다.

- 캐릭터: `@캐릭터명`. 예: `@CaptainZoro walks through a futuristic city.`
- 아바타(본인): `@me`. 본인 모습·목소리를 반영한다
- 시각 소재(ingredients): 모델 → 동영상 → 소재에 이미지·영상을 넣고, 프롬프트에서 `@애셋명`으로 부른다
- 시작/종료 프레임: 모델 → 동영상 → 프레임에 이미지를 넣는다
- 음성 참조: Omni Flash → 동영상 → 소재 → 음성. 프롬프트에 `@Voice: 이름`. 소재를 쓰는 동영상에서만 가능하다(그 외 생성에 넣으면 오류)

## 이미지 → 영상 워크플로우

캐릭터·구도를 이미지로 먼저 확정한 뒤 영상화하면 일관성과 통제력이 올라간다. Flow는 이미지 생성(Nano Banana 2/Pro, Imagen 4)과 영상 생성이 한 프로젝트에서 이어진다. 사용자가 "이미지 만들고 그걸로 영상", "캐릭터부터 만들어서 영상화" 같은 요청을 하면 2단계로 만든다.

### 1단계 — 이미지 프롬프트

영상의 첫 프레임이 될 정지 이미지를 만든다. 5형식 중 움직임(Action, 카메라 무빙, 사운드)을 빼고 정지 화면에 필요한 항목만 영어 단락으로 적는다.

```
A small white Maltese dog wearing a tiny white chef's hat, standing at a forest
campsite beside a stone campfire at dusk. Warm campfire glow on the dog's face,
soft blue dusk light. Cinematic 35mm film look, shallow depth of field. Plain
composition, the dog centered.
```

- 캐릭터·오브젝트를 소재로 재사용할 거면 단색 또는 분리된 배경에 둔다. 배경이 깔끔해야 모델이 잘 합성한다.
- 화면비는 영상과 동일하게 맞춘다.

### 2단계 — 영상 프롬프트

만든 이미지를 영상 입력으로 넣는 방법은 두 가지다.

| 방법 | 쓰는 때 | Flow 조작 |
|------|---------|-----------|
| 프레임(시작/종료) | 정한 이미지에서 영상이 시작·끝나게 | 모델 → 동영상 → 프레임 |
| 소재(참조) | 캐릭터·오브젝트를 여러 클립에 일관되게 | 모델 → 동영상 → 소재 |

프레임 방식은 프레임 사이 동작·전환만 적는다. 화면에 이미 있는 외형은 다시 묘사하지 않는다. 소재 방식은 5형식을 쓰되 어떤 이미지를 소재로 넣는지 명시한다.

```
(프레임 방식)
Slow dolly in. The dog stirs the pot with a wooden spoon, then looks up at the
camera cheerfully. A cheerful, bright voice says, "오늘 메뉴는 스튜입니다."
SFX: campfire crackle.
```

- 텍스트는 이미지와 모순되면 안 된다. 보완만 한다.
- 음성 참조는 소재를 쓰는 영상에만 붙는다. 프레임만 쓰는 영상에 음성 참조를 넣으면 오류가 난다.
- 1단계·2단계와 프레임/소재 중 무엇을 골랐는지 해설에 밝힌다.

## 아바타 립싱크·음성 추가 (이 스킬의 주 용도)

출처: support.google.com/flow 아바타·동영상 문서.

### 아바타와 캐릭터를 구분한다

- 아바타(`@me`): 본인 모습·목소리를 반영한다. 컴퓨터에서 시작하지만 휴대기기로 QR을 스캔해 캡처를 완료한다. EEA·영국·스위스에서는 아바타를 사용할 수 없다. 그 외 지역은 사용 가능하다.
- 캐릭터(`@캐릭터명`)·소재: 참고 이미지로 만든 인물·오브젝트를 여러 클립에 일관되게 쓴다. 본인이 아닌 캐릭터의 외형 고정은 이쪽이다.

### 대사는 따옴표로

립싱크는 대사를 따옴표로 적어야 음성과 입 모양이 함께 생성된다. 누가 말하는지도 명시한다.

```
The avatar looks into the camera and says, "안녕하세요, 오늘 영상을 시작합니다." SFX: soft room ambience.
```

### 음성 톤·음성 참조·커스텀 음성

- 음성 톤은 화자 묘사로 적는다. 예: `A warm, calm female voice in her 30s says, "환영합니다."` 속성: 성별, 연령대, 분위기(warm/energetic/calm/serious), 속도.
- 여러 클립에서 같은 목소리를 유지하려면 음성 참조를 쓴다. Omni Flash → 동영상 → 소재 → 음성에서 음성을 고르고 `@Voice: 이름`으로 부른다. 소재를 쓰는 동영상에서만 가능하다.
- 커스텀 음성은 Omni Flash → 음성 → 새 음성 만들기에서 사전 설정 음성을 고르고 스타일을 말로 설명해 만든다. 예: "뉴욕 억양을 살려 약간 허스키하게."

### 대사 길이와 한국어 처리

- 립싱크는 대사가 짧을수록 정확하다. 8초 클립에는 한 문장(약 15~20자)이 적정하다. 길면 샷을 나눠 한 문장씩 배분한다.
- 한국어 대사는 그대로 적으면 한국어 립싱크를 시도한다. 발음이 까다로운 문장은 짧게 끊는다. 의성어성 소리는 대사가 아니라 `SFX:`로 영어 묘사한다.

## 5형식 프롬프트 출력 형식

블록 헤더는 영어로 고정한다. 정보가 없는 항목은 생략한다(억지로 채우지 않는다).

```
[Cinematography] 샷 크기·앵글·무빙.
[Subject]        주체와 외형. 누가/무엇이.
[Action]         주체의 동작·반응.
[Context]        장소·배경·시간대.
[Style & Ambiance] 미감·무드·조명.
Dialogue: 대사는 따옴표로.
SFX / Ambient: 효과음·주변음.
Negative: 제외할 요소 서술.
```

라벨 스캐폴드를 먼저 내고, 이어서 라벨을 뗀 산문 한 문단을 낸다. Flow에는 산문형을 입력한다.

## 샷 분할 출력

한 클립 길이를 넘는 묘사는 샷별로 나눠 번호를 매긴다. 샷을 이어 붙이는 건 Flow 장면 빌더로 한다(프로젝트당 한 장면, 클립 순서 정렬·재정렬·트림). 길이를 늘릴 땐 "연장" 기능도 쓴다.

```
[Shot 1 / 8s]
[Cinematography] ...
...

[Shot 2 / 8s]
[Cinematography] ...
...
```

샷 사이 연결이 필요하면 "Shot 2는 Shot 1의 마지막 프레임에서 이어진다"를 해설에 적는다. 대사가 길어 나눈 경우 각 샷에 한 문장씩 배분했음을 밝힌다.

## 영상 편집 (Gemini Omni Flash)

기존 영상을 말로 고치는 작업은 Omni Flash가 맡는다. Flow에서 클립을 고르고 아래 모드를 쓴다.

- 연장: 클립 끝에 영상을 더한다. 연장 클립엔 삽입·삭제·카메라 수정을 적용할 수 없다.
- 삽입/삭제: 화면에서 상자를 그려 객체를 넣거나 지운다.
- 카메라: 카메라 움직임·위치를 바꾼다.
- 프레임을 이미지로 저장해 다음 영상의 소재·시작·종료 프레임으로 재사용한다.
- 수정해도 원본은 남는다. 기록 패널에 이전 버전과 프롬프트가 보존된다.

편집 프롬프트는 무엇을 바꿀지 한 번에 하나씩 지시한다. 입력 자산을 먼저 명시하고("이 영상의 배경만 바꿔"), 바꿀 내용을 적는다.

## 에이전트

Flow에는 대화형 공동작업자인 에이전트가 있다(웹·PC 한정). 브레인스토밍, 프롬프트 다듬기, 생성, 애셋 수정, 일괄 생성, 애셋 정리를 대화로 시킨다. 에이전트 쿼리 자체는 크레딧을 쓰지 않지만(일일 할당량 있음), 생성한 미디어는 크레딧을 쓴다. 에이전트에 "요청 사항"(가이드라인·참고 이미지)을 걸어 프로젝트 전체의 동작을 일관되게 만들 수 있다. 사용자가 "변형 5개 만들어 줘" 같은 일괄 요청을 하면 에이전트 활용을 권한다.
